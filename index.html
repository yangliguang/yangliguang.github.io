Java中synchronized详解

发表于 2016-05-31   |  

Java关键字，当它修饰一个方法或者代码块时，能够保证在同一时刻最多只有一个线程执行该段代码。

  1. 当两个并发线程访问同一个对象object中的这个synchronized(this)同步代码块时，一个时间内只能有一个线程得到执行。另一个线程必须当前线程执行完这个代码块以后才能执行该代码块。

  2. 然而，当一个线程访问object的一个synchronized(this)同步代码块时，另一个线程仍然可以访问该object中的非synchronized(this)同步代码块。

  3. 尤其关键的是，当一个线程访问object的一个synchronized(this)同步代码时，其它线程对object中所有其它synchronized(this)同步代码块的访问将被阻塞。

  4. 第三个例子同样适用于其它同步代码块。也就是说，当一个线程访问object的一个synchronize(this)同步代码块时，它将获得这个object的对象锁。结果，其它线程对该object对象所有同步代码部分的访问将被暂时阻塞。

  5. 以上规则对其它对象锁同样适用。

Java中static详解

发表于 2016-05-31   |  
  1. static变量

按照是否是静态，对类成员变量进行分类：一是static修饰的，叫静态变量或类变量，二是没有static修饰的，叫实例变量。两者的区别在于：

静态变量在内存中只有一个拷贝(节约内存)，JVM只为静态变量分配一次内存，在加载类的过程中完成静态变量的内存分配，可用类名直接访问(方便)，当然也可以通过对象来访问(不推荐)。对于实例变量，每创建一个实例，JVM就会为实例变量分配一次内存，实例变量可以在内存中有多个拷贝，互不影响(灵活)。

  1. static方法

静态方法可以通过类名调用，任何实例也可以调用，因此静态方法中不能用this和super关键字，不能直接访问所属类的实例变量和实例方法(即常见的静态方法不允许访问类的非静态方法)，只能访问所属类的静态变量和静态成员方法，因为static方法独立于任何实例，因此static方法必须被实现，而不能是抽象的(这也是static不能和abstract共用的原因)。

  1. static代码块

也叫静态代码块，在类中独立于类成员的static语句块，可以有多个，位置随意放，JVM加载类的时候会执行，如果有多个，则按照在类中出现的先后顺序依次执行，仅执行一次。

  1. static和final共用

static和final修饰成员变量和成员方法，可以简单理解为“全局变量”！

对于变量，表示一旦给值不可修改，并且通过类名可以直接访问。

对于方法，表示不可覆盖，并且可以通过类名直接访问。

Javaz中final详解

发表于 2016-05-30   |  
  1. final变量

变量一旦被初始化便不可改变(对基本类型来说，值不可变，对数组和对象来说，引用不可变)，初始化可以在两个地方，一是定义处，二是构造方法中，两者二选一。

当函数参数为final类型时，可以读取该参数，但无法修改。

当方法中的内部类在用到方法的参变量时，此参变也必须声明为final才可使用。

  1. final方法

如果一个类不允许其子类覆盖某个方法，则可以把该方法声明为final方法。

使用final方法原因有二:

第一、把方法锁定，防止任何继承类修改它的意义和实现。

第二、高效。编译器在遇到调用final方法时会转入内嵌机制，大大提高执行效率。

  1. final类

final类不能被继承，因此final类的成员方法没有机会被覆盖，默认都是final的。在设计类的时候，如果不需要其有子类，类的实现细节不允许改变，并且确信该类不会被扩展，那么设计为final类。

Java中abstract详解

发表于 2016-05-30   |  

Abstract可以修饰类、方法

修饰类，则该类必须被继承(与final永不会同时出现)，不可被实例化成对象。

和接口以及其它基类一样，可以声明，作为编译时类型，但不能作为运行时类型。

修饰方法时，该方法为抽象方法，不需要实现，留给子类覆盖，子类覆盖后该方法才能生效。

注意比较：

  • private void print(){}；此语句表示方法的空实现。

  • Abstract void print()； 此语句表示方法的抽象，无实现。

如果一个类中有一个抽象方法，则该类一定是抽象类，反之，一个抽象类，可能有抽象方法。

为了实现多态，父类必须有定义，而不实现，留给子类实现，此时可将父类定义为abstract类。

最后一点，修饰方法时，abstract与statci不能放一起，因为static不可被覆盖吗，而abstract生来就是为了被覆盖。

markdown的超链接加上target='_blank'

发表于 2016-05-18   |  

Markdown支持两种形式的链接语法：行内式和参考式两种形式。
不管是哪一种，链接文字都是用 [方括号] 来标记。

要建立一个行内式的链接，只要在方块括号后面紧接着圆括号并插入网址链接即可，
如果你还想要加上链接的title文字，只要在网址后面，用双引号把title文字包起来即可，例如：

This is [an example](http://example.com/ title="Title") inline link.

会产生

<p>This is <a href="http://example.com/" title="Title">an example</a> inline link.</p>

参考式的链接是在链接文字的括号后面再接上另一个方括号，而在第二个方括号里面要填入用以辨识链接的标记：

This is [an example][id] reference-style link.

接着，在文件的任意处，你可以把这个标记的链接内容定义出来：

[id]: http://example.com/  "Optional Title Here"

而我们打开所生产的超链接，默认是在本窗口打开的，为了有更好的阅读体验，我们往往希望在新窗口
打开超链接，并不希望影响阅读本文。markdown目前应该还不支持这种语法的，
但可以用其他方式来解决 比如jQuery 在合适的地方加上如下代码：

<script type="text/javascript">
    $(document).ready(function() {
        //为超链接加上target='_blank'属性
        $('a[href^="http"]').each(function() {
            $(this).attr('target', '_blank');
        });
    });
</script>
