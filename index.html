Java中的反射机制

发表于 2016-06-01   |  

两大类功能：

  • 通过对象获取完整的类信息，父类信息，接口信息，类的加载器信息

  • 所有的对象都是Class类的实例，通过类名实例化类的对象，调用方法

常见情景：

  • 通过反射调用类的方法

  • 操作对象的属性

  • 修改数组信息

  • 获取类加载器

  • 用于工厂模式

类的生命周期

在一个类编译完成之后，下一步就需要开始使用类，如果要使用一个类，肯定离不开JVM。在程序执行中JVM通过装载，链接，初始化这3个步骤完成。

类的装载是通过类加载器完成的，加载器将.class文件的二进制文件装入JVM的方法区，并且在堆区创建描述这个类的java.lang.Class对象。用来封装数据。 但是同一个类只会被类装载器装载以前

链接就是把二进制数据组装为可以运行的状态。

链接分为校验，准备，解析这3个阶段

校验一般用来确认此二进制文件是否适合当前的JVM（版本），

准备就是为静态成员分配内存空间，。并设置默认值

解析指的是转换常量池中的代码作为直接引用的过程，直到所有的符号引用都可以被运行程序使用（建立完整的对应关系）

完成之后，类型也就完成了初始化，初始化之后类的对象就可以正常使用了，直到一个对象不再使用之后，将被垃圾回收。释放空间。

当没有任何引用指向Class对象时就会被卸载，结束类的生命周期

线性时间排序

发表于 2016-06-01   |  

序言

之前接触的冒泡排序，快速排序，归并排序，堆排序等算法都存在比较，称为“比较排序”，时间复杂度下界是O(nlogn)，接下来介绍几种线性(O(n))排序算法。

计数排序

引出

过程描述：假设待排序序列a中值的范围[0,k],其中k表示待排序序列中的最大值。首先用一个辅助数组count记录各个值在a中出现的次数，比如count[i]表示i在a中的个数。然后依次改变count中元素值，使count[i]表示a中不大于i的元素个数。然后从后往前扫描a数组，a中的元素根据count中的信息直接放到辅助数组b中。最后把有序序列b复制到a。

代码

package sort;

/**
 * Author:Young
 * Class Comment:
 * Date: 2016年6月2日下午6:58:17
 */
public class CountingSort {
    public static void main(String[] args) {
        int[] nums = new int[10];
        for(int i = 0; i < 10; i++){
            nums[i] = (int)(Math.random() * 10) + 90;
        }
        for(int i : nums)
            System.out.print(i+" ");
        System.out.println();
        sort(nums, 100);
        for(int i : nums)
            System.out.print(i+" ");
    }

    public static void sort(int[] nums, int max){
        int[] count = new int[max+1];
        for(int i : nums)
            count[i]++;
        for(int i = 0, j = 0; i < count.length; i++){
            while(count[i] > 0){
                nums[j++] = i;
                count[i]--;
            }
        }
    }
}

输出结果：

93 99 94 94 91 94 95 93 95 95 
91 93 93 94 94 94 95 95 95 99

效率分析

从代码看，一个one pass的for循环，O(n),一个for和while的嵌套循环，O(max+n),时间复杂度不变，可见计数排序是稳定的，辅助空间max. 有三个约束条件：

  • 数组取值范围[0,max]
  • 元素值需是非负数
  • max的值不能太大

Java中synchronized详解

发表于 2016-05-31   |  

Java关键字，当它修饰一个方法或者代码块时，能够保证在同一时刻最多只有一个线程执行该段代码。

  1. 当两个并发线程访问同一个对象object中的这个synchronized(this)同步代码块时，一个时间内只能有一个线程得到执行。另一个线程必须当前线程执行完这个代码块以后才能执行该代码块。

  2. 然而，当一个线程访问object的一个synchronized(this)同步代码块时，另一个线程仍然可以访问该object中的非synchronized(this)同步代码块。

  3. 尤其关键的是，当一个线程访问object的一个synchronized(this)同步代码时，其它线程对object中所有其它synchronized(this)同步代码块的访问将被阻塞。

  4. 第三个例子同样适用于其它同步代码块。也就是说，当一个线程访问object的一个synchronize(this)同步代码块时，它将获得这个object的对象锁。结果，其它线程对该object对象所有同步代码部分的访问将被暂时阻塞。

  5. 以上规则对其它对象锁同样适用。

Java中static详解

发表于 2016-05-31   |  
  1. static变量

按照是否是静态，对类成员变量进行分类：一是static修饰的，叫静态变量或类变量，二是没有static修饰的，叫实例变量。两者的区别在于：

静态变量在内存中只有一个拷贝(节约内存)，JVM只为静态变量分配一次内存，在加载类的过程中完成静态变量的内存分配，可用类名直接访问(方便)，当然也可以通过对象来访问(不推荐)。对于实例变量，每创建一个实例，JVM就会为实例变量分配一次内存，实例变量可以在内存中有多个拷贝，互不影响(灵活)。

  1. static方法

静态方法可以通过类名调用，任何实例也可以调用，因此静态方法中不能用this和super关键字，不能直接访问所属类的实例变量和实例方法(即常见的静态方法不允许访问类的非静态方法)，只能访问所属类的静态变量和静态成员方法，因为static方法独立于任何实例，因此static方法必须被实现，而不能是抽象的(这也是static不能和abstract共用的原因)。

  1. static代码块

也叫静态代码块，在类中独立于类成员的static语句块，可以有多个，位置随意放，JVM加载类的时候会执行，如果有多个，则按照在类中出现的先后顺序依次执行，仅执行一次。

  1. static和final共用

static和final修饰成员变量和成员方法，可以简单理解为“全局变量”！

对于变量，表示一旦给值不可修改，并且通过类名可以直接访问。

对于方法，表示不可覆盖，并且可以通过类名直接访问。

Javaz中final详解

发表于 2016-05-30   |  
  1. final变量

变量一旦被初始化便不可改变(对基本类型来说，值不可变，对数组和对象来说，引用不可变)，初始化可以在两个地方，一是定义处，二是构造方法中，两者二选一。

当函数参数为final类型时，可以读取该参数，但无法修改。

当方法中的内部类在用到方法的参变量时，此参变也必须声明为final才可使用。

  1. final方法

如果一个类不允许其子类覆盖某个方法，则可以把该方法声明为final方法。

使用final方法原因有二:

第一、把方法锁定，防止任何继承类修改它的意义和实现。

第二、高效。编译器在遇到调用final方法时会转入内嵌机制，大大提高执行效率。

  1. final类

final类不能被继承，因此final类的成员方法没有机会被覆盖，默认都是final的。在设计类的时候，如果不需要其有子类，类的实现细节不允许改变，并且确信该类不会被扩展，那么设计为final类。

Java中abstract详解

发表于 2016-05-30   |  

Abstract可以修饰类、方法

修饰类，则该类必须被继承(与final永不会同时出现)，不可被实例化成对象。

和接口以及其它基类一样，可以声明，作为编译时类型，但不能作为运行时类型。

修饰方法时，该方法为抽象方法，不需要实现，留给子类覆盖，子类覆盖后该方法才能生效。

注意比较：

  • private void print(){}；此语句表示方法的空实现。

  • Abstract void print()； 此语句表示方法的抽象，无实现。

如果一个类中有一个抽象方法，则该类一定是抽象类，反之，一个抽象类，可能有抽象方法。

为了实现多态，父类必须有定义，而不实现，留给子类实现，此时可将父类定义为abstract类。

最后一点，修饰方法时，abstract与statci不能放一起，因为static不可被覆盖吗，而abstract生来就是为了被覆盖。
